HTML

  • HTML elements work together to make the framework of a webpage
  • The head element contains information about the webpage
  • The head element is a container for metadata. Meta elements contain information about the page that is used by the browser
  • The body element represents the visible content shown to the user
  • Elements that are nested inside other elements are known as child elements

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Inline CSS is used when a developer wants to style an element directly in the HTML file by applying a specific HTML tag
  • Most popular way to style a webpage is by using an external CSS style sheet
  • A selector defines the element or attributes to which the rules or declerations will apply
  • You can combine two selectors into a single CSS rule by using a comma in order to not repeat youself (DRY) code

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Always make a feature branch to work in as to maintain the main branch
  • The feature branch is where we make commits of all the changes made to the code. Always add a commit message describing what changed in the code
  • More info about Git version control in prework module 2.4.6

JavaScript

  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
  • You can use the "console.log()" method to output a message to the web console by adding an argument in the parentheses
  • When using "console.log()", always define the variable first before calling it to be logged to the console
  • Check prework module 3.3.4 for more information about conditional statements including the 'if' statement
  • An array is a single variable that is used to hold a group of data. Arrays are used to hold data that is related in some way
  • JavaScript arrays are zero-indexed and sequential. Meaning, the first item in an array has an index of 0, not 1
  • A 'for' loop uses the predictable pattern of indices to preform a task on all the items in an array by allowing a single code block to be executed over and over
  • Check prework module 3.3.5 for more information about using a 'for' loop
  • A function is a set of instructions that tells the computer how to preform a certain task
  • Functions do not automatically execute when the JavaScript file is run, you have to call a function by name in order to execute it
